Introduction

This technical documentation page will be dedicated to version control system git. It will help guide a newbie to quickly set up and understand how to use git via command line terminal.

Simply put, a version control is a local or remote system that can track, record, and recall the life-cycle of a file or a series of files. Git generally only ‘adds’ data, but don’t be fooled you can accidentally lose sight of your data if you improperly handle commits, branches, etc..

Git uses a file system called a repository that tracks within the folder you initiate git to track (more on that later).

Git Setup

Customizing your git pathway will avoid hazardous confusion and random path errors.

1. git config - is a powerful way to set up your user identity such as: user.name, user.email, colors, etc.

2. Choose a location on your hard drive where you want to store git configuration, after use this command:

$ [path] etc/gitconfig

For values specific personally to the user use this command instead:

~/.gitconfig or ~/.config/git/config

Your identity is how anyone knows who did what, but this also helps track your work, just in case you need to revert something back to a prior state.

    User.name

  • This will help identify yourself, although it would be advised to use your full name.
    • $ git config --global user.name "James Jeremiah King"

    User.email

  • Next add the email you want associated with git.
    • $ git config --global user.email jamesjking@example.com

    Final step in git config is to checkout your settings:

    $ git config --list --show-origin

Git Initiated

The great thing about git is that it can be added to an existing directory or initialized to start a new project.

Linux:

$ cd home/user/my_project

You will notice in that .git directory that magically appeared, this is a new sub-directory that contains your essential git repository files.

Say there is an amazing project that you want to get involved in either for work, or to gain experience, than you must copy/clone an existing repository.

Copying or Cloning a public or private repository

  • Simple cloning:
    • $ git clone https://github.com/jking026/freeCodeCamp.git

Git Committed

Congrats, if you made it this far, then you are truly committed to git, so what now?

When you are making changes, keep in mind that each file in your working directory can be in one of two states: tracked or untracked.

Tracked & Untracked files

  • Tracked files are files that git knows about, while untracked are everything else mainly that are not in the staging area or the last snapshot.
  • To check the status of your branch and its content use this:
    • $ git status

    With the git status command you can check which branch you are in, if your files are tracked/untracked , staged or unstaged.

    • When you want stage a modified file use this command:
      • $ git add
    • or to add all files
      • $ git add .

    Once they are staged you must commit them to the head of the repository, but a message is mandatory, if you don’t add a message-git will automatically abort your commit.

    $ git commit -m “blah blah blah”

    Git Help

    Doing your homework is essential, especially on the current version of git. There is a vast amount git content out there and most are based on particular versions. The best way to “git help” is to update to the most current version.

    Manual Man

    If you are finding yourself lost at anytime, refer to one of these terminal commands:

      $ man git-log [verb] $ git [verb] --help $ man git-[verb]
    • For more information about git log use this command:
      • $ git help log

    If you ever are to run into an issue, definitely initiate the git help or git man command.
